# KJH_EV4 — 전기차 고전압 배터리 화재 방재 시스템

ESP32-S3 기반의 **이동식 방수포 수조 자동 전개 제어 시스템**입니다.  
열화상 카메라(MLX90640)와 MQ 가스 센서를 융합 판정하여 배터리 팩의 열폭주(Thermal Runaway)를 조기 감지하고,  
TB6600 스텝 모터로 리프트를 구동해 침수 수조를 차량 위에 전개한 뒤 소화수 펌프를 순환 가동하여 화재를 진압합니다.

## 1. 시스템 개요

```
[열화상 카메라] ──┐
                   ├──► ESP32-S3 ──► TB6600 ──► 스텝 모터 ──► 리프트 (수조 전개)
[MQ 가스 센서] ──┘         │
                            └──► 릴레이 ──► 소화수 펌프
```

| 구분 | 내용 |
|------|------|
| **화재 감지** | MLX90640 열화상 × MQ 가스 센서 복합 판정 |
| **기구 제어** | TB6600 스텝 모터 드라이버 — 리프트 상승/하강 |
| **소화 수단** | 소화수 펌프 릴레이 — 수조 주수 및 순환 |
| **수동 제어** | 단일 비상 스위치 (짧게/길게 누름) |
| **MCU** | ESP32-S3 (4D Systems Gen4 R8N16, 16MB Flash / 8MB PSRAM) |

## 3. 핀맵 및 배선

### 3-1. GPIO 핀맵

| 기능 | GPIO | 방향 | 신호 레벨 | 연결 대상 |
|------|------|------|-----------|-----------|
| 스텝 모터 펄스 (PUL+) | **GPIO 4** | 출력 | 3.3V PWM | TB6600 PUL+ |
| 스텝 모터 방향 (DIR+) | **GPIO 5** | 출력 | 3.3V | TB6600 DIR+ |
| 비상 스위치 | **GPIO 6** | 입력 (풀업) | Active-LOW | 푸시버튼 → GND |
| 소화수 펌프 릴레이 | **GPIO 8** | 출력 | Active-HIGH | 릴레이 모듈 IN |
| UART2 RX | **GPIO 17** | 입력 | 3.3V UART | GY-MCU90640 TX |
| UART2 TX | **GPIO 18** | 출력 | 3.3V UART | GY-MCU90640 RX |
| MQ 가스 센서 (ADC1 CH6) | **GPIO 7** | 입력 | 0~3.3V ADC | MQ 센서 AO 핀 |

### 3-2. TB6600 배선

| TB6600 단자 | 연결처 | 비고 |
|-------------|--------|------|
| PUL+ | ESP32-S3 GPIO 4 | 3.3V 신호 |
| PUL- | GND | 공통 GND |
| DIR+ | ESP32-S3 GPIO 5 | 3.3V 신호 |
| DIR- | GND | 공통 GND |
| ENA+ | 5V 또는 개방 | 상시 활성화 |
| ENA- | GND | |
| A+, A- | 스텝 모터 코일 A | |
| B+, B- | 스텝 모터 코일 B | |
| VCC, GND | 24~48V DC 전원 | |

> **TB6600 전류/마이크로스텝 설정**: SW1~SW6 딥스위치로 설정.  
> 본 시스템은 **800 pulse/rev (마이크로스텝 1/4)** 기준입니다. 변경 시 `MOTOR_PULSES_PER_REV` 수정 필요.

### 3-3. 모터 방향 정의

| DIR 핀 레벨 | 모터 회전 | 리프트 동작 | 수조 동작 |
|-------------|-----------|-------------|-----------|
| `DIR = 0` (LOW) | 시계 방향 (CW) | **상승** | 수조가 위로 올라가 차량을 덮음 |
| `DIR = 1` (HIGH) | 반시계 방향 (CCW) | **하강** | 수조가 내려가 원위치로 복귀 |

## 4. 소프트웨어 동작 원리

### 4-1. FreeRTOS 태스크 구조

```
┌─────────────────────────────────────────────────────────┐
│  Core 1 (우선순위 3)   vTaskEmergencyMonitor             │
│  ─ 상태 머신 구동                                        │
│  ─ 리프트 모터 제어 (move_*) 및 펌프 릴레이 제어         │
│  ─ 화재 트리거 즉시 비상 전개                            │
└─────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────┐
│  Core 0 (우선순위 1)   vTaskSensorAndImageProcessing     │
│  ─ MLX90640 UART 프레임 수신 및 파싱                     │
│  ─ MQ 가스 센서 ADC 수집                                 │
│  ─ 융합 화재 판정 → hybrid_fire_triggered 플래그 설정    │
└─────────────────────────────────────────────────────────┘
```

### 4-2. 모터 제어 함수

| 함수 | 방향 | 펌프 동작 | 용도 |
|------|------|-----------|------|
| `move_reverse_with_mon()` | DIR=0 (상승) | 없음 | 평상시 수동 수조 전개 |
| `move_emergency_return()` | DIR=0 (상승) | 이동 거리 **1/3 지점에서 ON** | 화재 비상 전개 |
| `move_forward_return()` | DIR=1 (하강) | **즉시 OFF** (첫 줄) | 수조 원위치 복귀 |

#### 펌프 기동 타이밍 (`move_emergency_return` 2-Phase 구조)

```
스텝:  0 ──────── 1/3 ──────────────────── 끝
              ↑
         [루프 정지]
         100ms 대기
         릴레이 ON  ← EMI 간섭 방지를 위해 PUL 루프 완전 정지 후 전환
         100ms 대기
         [루프 재개]

펌프:  OFF ──────── ON ────────────────────────────────►
                         (화재 순환 중, 화재 해제 후에도 유지)
                                          ↓
                                 move_forward_return() 호출 시 즉시 OFF
```

> **EMI 간섭 방지**: 릴레이 코일의 역기전력이 TB6600 PUL 신호를 교란하여 모터가 멈추는 현상을 방지하기 위해,  
> 릴레이 전환은 반드시 스텝 펄스 루프가 완전히 정지된 상태에서 수행됩니다.

## 5. 상태 머신

```
               [ 부팅 ]
                  │
                  ▼
      ┌─────────────────────┐
      │  STATE_READY_REVERSE │  ← 상단 원점 대기
      │  (수조 수납 위치)    │
      └─────────────────────┘
          │          │          │
     짧게 누름    길게 누름    화재 감지
     (3초 미만)  (3초 이상)
          │          │          │
          ▼          ▼          ▼
     [리프트 하강]  [강제 상단  [비상 전개]
     move_reverse   복귀]       move_emergency_return()
     _with_mon()  move_forward  └─ 1/3 지점에서 펌프 ON
          │        _return()    └─ 화재 순환 루프 진입
          │                         │
    중도 버튼    완주                │ 화재 해제 + 버튼
          │          │              │
          ▼          ▼              ▼
     [즉시 복귀]  STATE_READY  [강제 상단 복귀]
     move_forward  _FORWARD    move_forward_return()
     _return()     (하단 감시) └─ 펌프 즉시 OFF
                    │
               버튼 누름
                    │
                    ▼
             [상단 복귀]
             move_forward_return()
             └─ 펌프 즉시 OFF
```

## 6. 화재 판정 알고리즘

### 6-1. 융합 판정 로직

```
MQ 가스 센서값 (ADC 0~4095)
    │
    ├─ raw ≥ 1200 또는 (300~599)  →  임계온도 40°C,  최소면적 4 px  (고위험)
    ├─ raw 600~1199               →  임계온도 65°C,  최소면적 8 px  (중위험)
    └─ raw < 300                  →  임계온도 75°C,  최소면적 12 px (기본)

MLX90640 32×24 = 768 픽셀 온도 매트릭스
    │
    └─ [임계온도 이상 픽셀 수] ≥ [최소면적] ?
           YES → hybrid_fire_triggered = true
           NO  → hybrid_fire_triggered = false
```

### 6-2. UART 프레임 파싱 (GY-MCU90640)

- 프레임 크기: **1544 바이트** (헤더 `0x5A 0x5A` + 4바이트 헤더 + 1536바이트 온도 + 4바이트 트레일러)
- 픽셀 데이터: 리틀 엔디안 16비트 정수, 실제 온도 = raw ÷ 100 (°C)
- 유효 범위 필터: `5.0°C ~ 120.0°C` (노이즈 픽셀 제거)
- 버퍼 과부하 방지: 4000 바이트 이상 적체 시 `uart_flush_input()` 자동 수행

## 7. 주요 파라미터 설정

`src/main.c` 상단 `#define` 에서 조정합니다.

| 매크로 | 기본값 | 설명 |
|--------|--------|------|
| `MOTOR_PULSES_PER_REV` | `800` | TB6600 마이크로스텝 설정값 (1/4 스텝 기준) |
| `MOTOR_TARGET_REV` | `20.0` | 리프트 유효 이동 거리 (스텝 모터 총 회전 수) |
| `MOTOR_PULSE_DELAY_US` | `400` | 스텝 펄스 간격 μs (줄이면 빠름, 늘리면 토크↑) |
| `BUTTON_RESET_HOLD_MS` | `3000` | 강제 원위치 복귀 롱프레스 판정 시간 (ms) |
| `TEMP_FIRE_PIXEL` | `75.0` | 기본 화재 판정 픽셀 온도 (°C) |
| `FIRE_MIN_AREA_PIXELS` | `12` | 화재 확정 최소 픽셀 수 |
| `MQ_FIRE_THRESHOLD` | `1200` | MQ 센서 고위험 ADC 임계값 |

> **`max_target_steps`** = `MOTOR_PULSES_PER_REV × MOTOR_TARGET_REV` = 16,000 스텝 (기본값)  
> 리프트 기구부 변경 시 `MOTOR_TARGET_REV` 값으로 이동 거리를 조정하세요.

## 8. 빌드 및 플래시

### 사전 요구사항

- [PlatformIO](https://platformio.org/) (VSCode 확장 or CLI)
- ESP-IDF 프레임워크 (PlatformIO가 자동 설치)

```bash
# 의존성 설치 및 빌드
pio run

# 펌웨어 업로드 (USB 연결 필요)
pio run -t upload

# 시리얼 모니터 (115200 baud)

```

```ini
; platformio.ini 에 추가
upload_port = /dev/ttyUSB0      ; Linux
upload_port = /dev/cu.usbmodem* ; macOS
upload_port = COM3               ; Windows
```

## 9. PC 열화상 실시간 모니터 (`tools/thermal_monitor.py`)

ESP32-S3를 USB로 PC에 연결하면 시리얼로 전송되는 `$DATA` 라인을 파싱하여  
**실시간 열화상 시각화 + CSV 자동 저장**을 수행하는 Python 스크립트입니다.

### 설치

```bash
cd tools
pip install -r requirements.txt
```

### 실행

```bash
# 포트 목록 확인
python thermal_monitor.py --list

# 실행 (포트 지정)
python thermal_monitor.py --port COM3                     # Windows
python thermal_monitor.py --port /dev/ttyUSB0             # Linux
python thermal_monitor.py --port /dev/cu.usbmodem1101     # macOS

# CSV 저장 없이 시각화만
python thermal_monitor.py --port COM3 --no-csv
```

### 화면 구성

```
┌──────────────────┬────────────────┬──────────────┐
│                  │  최고온도 시계열 │              │
│  MLX90640        │  (실시간 그래프) │  상태 패널   │
│  32×24 열화상    ├────────────────┤              │
│  (inferno 컬러맵)│  MQ 가스센서   │  SAFE / FIRE │
│                  │  (실시간 그래프) │  수치 현황   │
└──────────────────┴────────────────┴──────────────┘
```

- 열화상: 최고온도 위치에 시안색 십자선 자동 표시, 컬러맵 범위 자동 조정
- 상태 패널: 화재 감지 시 빨간색 `FIRE !!` 표시 및 콘솔 경보 출력
- 시계열: 최근 300 샘플 표시, 화재 기준선 / 가스 경계선 오버레이

### CSV 저장 형식

`tools/logs/thermal_YYYYMMDD_HHMMSS.csv` 로 자동 저장됩니다.

| 컬럼 | 내용 |
|------|------|
| `timestamp` | PC 로컬 시각 (ms 단위) |
| `tick_ms` | ESP32 FreeRTOS 틱 카운터 (ms) |
| `mq_raw` | MQ 가스 센서 ADC 값 |
| `max_temp_c` | 프레임 최고 온도 (°C) |
| `hot_pixels` | 임계 온도 이상 픽셀 수 |
| `fire_triggered` | 화재 판정 (0/1) |
| `t000` ~ `t767` | 32×24 = 768개 픽셀 온도 (°C) |

### $DATA 프로토콜

ESP32가 500ms마다 출력하는 1줄 CSV 포맷:

```
$DATA,<tick_ms>,<mq_raw>,<max_temp>,<hot_pixels>,<fire>,<t0>,<t1>,...<t767>
```

## 12. 주의사항 및 알려진 제약

### 안전 주의사항

- **고전압 배터리 환경**: 이 시스템은 고전압(400V+) 배터리 팩 인근에 설치됩니다. 배선 절연 및 방수 처리를 철저히 하세요.
- **소화수 감전 위험**: 주수 전 차량 전원 차단 여부를 확인하거나, 전기 절연 소화 매질(탄산칼륨 수용액 등) 사용을 검토하세요.
- **릴레이 역기전력**: 릴레이 코일 양단에 **플라이백 다이오드**를 반드시 설치하세요. 미설치 시 ESP32 GPIO 손상 위험이 있습니다.

### 소프트웨어 제약

- `move_reverse_with_mon()` / `move_emergency_return()` / `move_forward_return()` 은 **블로킹 함수**입니다. 호출 중 다른 상태 전환은 불가합니다.
- `hybrid_fire_triggered` 는 `volatile bool`로 선언되어 있으며, 크로스코어 접근 시 `asm volatile("memw")` 메모리 배리어로 가시성을 보장합니다.
- 화재 감지 후 펌프는 **`move_forward_return()` 호출 시에만 OFF**됩니다. 전원 재시작으로는 릴레이 상태가 초기화되지 않으니 `init_hardware()` 내 `gpio_set_level(PIN_COOLING_PUMP, 0)` 초기화를 확인하세요.
- MQ 센서는 부팅 후 **예열 시간(약 2분)** 이 필요합니다. 예열 전에는 ADC 값이 불안정할 수 있습니다.

### 하드웨어 제약

- TB6600 DIR 핀 변경 후 최소 **200ms** 안정화 대기가 코드에 내장되어 있습니다 (`vTaskDelay(pdMS_TO_TICKS(200))`).
- 스텝 모터 실속(stall) 감지 기능은 없습니다. 기구부 끝단에 물리적 리밋 스위치 추가를 권장합니다.
